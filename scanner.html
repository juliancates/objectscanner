3 — Trace settings
Threshold Otsu auto — finds the best global threshold automatically. Best for most scans with clear contrast.

Adaptive — adjusts threshold per region; use for uneven lighting or faint pencil marks.

Fixed — set a specific brightness cutoff manually.
Fixed value128 Brightness cutoff (0–240). Pixels darker than this become black ink; lighter ones become white. Lower = only darkest marks; higher = includes lighter lines.
Smooth0.35 How much contour corners are softened using Catmull-Rom → cubic bezier conversion.

0 = angular straight lines. 1 = maximum smoothing. 0.35 is a good default for hand-drawn marks.
Simplify0.5 Ramer–Douglas–Peucker epsilon — maximum deviation allowed when reducing contour points.

0 = keep every pixel (large SVG). 3 = heavy reduction. 0.5 balances fidelity and file size.
Min area80 Contours smaller than this area (px²) are discarded as noise.

Increase to remove paper texture or grid artefacts. Decrease if small marks are missing from the output.

How it works
1. Print the mat — US Letter sheet with four unique binary corner markers at known positions.
2. Photo — Place your object on the mat and photograph from any angle; all four corners must be visible.
3. Homography — OpenCV computes a 3×3 perspective transform mapping your four clicked corners to a flat rectangle.
4. Warp — The image is rectified to perfect top-down view at the correct 8.5:11 aspect ratio.
5. SVG trace — The warped image is thresholded and contours are traced to SVG <path> elements using Ramer–Douglas–Peucker simplification. Output is infinitely scalable.
